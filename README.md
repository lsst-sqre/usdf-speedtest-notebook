usdf-speedtest-notebook
-----------------------

curl-speedtest.ipynb
====================

This is just a little notebook to fetch a large (1.3GB) file from USDF S3 and discard it.

It uses `curl` to pull (this is faster, empirically, than a Python HTTP client), and also tests multiprocess concurrency (using an asyncio TaskGroup, each task of which spawns a process with asyncio.subprocess).
We ran with pods with 4CPUs and 16GB of memory, but the memory and CPU actually used were trivial; the vast majority of the process time was spent waiting on network I/O.

It writes its output to a shared directory, and then we can plot that data.

We run it under mobu with a parallel flock; the largest we did was to run 100 pods wide (which resulted in 5 nodes) for a little over two hours.

USDF-speed-generic.ipynb
========================

This is the notebook that consumes the JSON data generated by the speedtest and plots it.
It shows both the results on a per-node basis and aggregated across nodes.
In general, the mean and median throughput is between 935-940MBps in our tests, with peaks nearing 2000MBps.  Perhaps if we discarded the tails where not all of the pods had started, or some had already finished, the mean and median would be more like 1000MBps.
